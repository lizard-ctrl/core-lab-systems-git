Chapter 04

Staging Area

A snapshot of changes you are about to commit to your repository

git add

Add changes in files that you choose to the staging area. This is a necessary step before committing changes.

git commit

Create a commit (a snapshot of the code base) from the changes that are in the staging area. Committing is how you you save the changes you made to the code in a particular point in time. Every commit has a unique identifier in the form of a hash.
