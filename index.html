Git Introduction

Chapter 09

While working on a project

  1. Get most recent changes - git pull
  2. Work on your code
  3. Very frequently, after you've completed a small change that you're pleased with:
    • Stage the changes to be committed - git add path/to/file path/to/another/file
    • Commit the changes - git commit
    • Type a descriptive message describing the changes you've made, save, and close the file.
  4. Sync changes to remote repository if you are using one (in this class we will host remote repositories on Github) - git push
